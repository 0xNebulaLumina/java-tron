Phase 3 – Java-tron ⇄ Rust Execution Module Integration & Verification
────────────────────────────────
Goal: Wire java-tron to the Rust Execution Module and prove functional equivalence to the legacy Java EVM before production cut-over.

A. Verification Strategy  
1. Shadow (Dual) Execution  
   • New config flag ExecutionMode = EMBEDDED | REMOTE | SHADOW (default = EMBEDDED).  
   • SHADOW mode executes every block in both engines, then:  
     – Compute a deterministic StateDigest on the modified-account set.  
     – Compare full receipts (status, gas used, bloom, logs).  
     – Panic or log on mismatch (configurable).  

2. StateDigest Definition  
   • For each modified account: address ‖ balance ‖ nonce ‖ codeHash ‖ sorted(storageSlots).  
   • Bytes concatenated → SHA-256 → per-account hash.  
   • XOR-reduce (or hash-chain) all per-account hashes ⇒ 32-byte digest.  
   • Implemented once in Rust; exposed to Java via JNI and/or a thin gRPC helper to guarantee identical output.

3. Historical Replay  
   • CLI tool replays the most-recent N main-net blocks (N is CLI arg; default 100 k).  
   • Stops on first digest/receipt mismatch and dumps diff details.

4. Golden Vector Suite  
   • Curated deterministic TX cases (contract creation, self-destruct, tricky precompiles, edge-gas).  
   • Stored as JSON; both engines must produce identical receipts & digests.

5. Continuous Canary  
   • Launch a test-net node in SHADOW mode.  
   • Alert on metric state_mismatch_total > 0.  
   • One full week of zero mismatches is the cut-over criterion.

B. Implementation Roadmap & Exit Criteria  
(Tasks are now tracked in the shared TODO list—IDs in brackets)

1. SPI Extension & Config Flags [phase3_spi_extension]  
   – Add ExecutionMode enum and env/property parsing.  
   – Exit: Node starts in all three modes with correct routing.

2. ExecutionGrpcClient [phase3_grpc_client]  
   – PickFirst LB provider registration, deadlines, retries.  
   – Exit: Single TX executes successfully via Rust engine.

3. StateDigest Utility [phase3_state_digest]  
   – Rust lib + JNI binding; cross-language test vectors pass.  
   – Exit: Same input → identical 32-byte digest in both languages.

4. Rust Snapshot Hooks [phase3_rust_snapshot_hooks]  
   – StorageAdapter emits modified-account set cheaply.  
   – Exit: Hook returns correct list with negligible overhead.

5. Shadow Harness in java-tron [phase3_shadow_harness]  
   – Captures pre/post snapshots, receipts, compares, handles config flags.  
   – Exit: Main-net block height X runs with 0 mismatches.

6. Golden Vector Suite [phase3_golden_vectors]  
   – 50+ edge-case TXs, automated JUnit tests.  
   – Exit: All vectors pass on CI.

7. Historical Replay Tool [phase3_historical_replay]  
   – Command‐line selectable N; progress logging; diff dump.  
   – Exit: Replays default 100 k blocks < X hours, 0 mismatches.

8. CI Integration [phase3_ci_integration]  
   – GitHub Action runs 10-block shadow test on every PR.  
   – Exit: CI fails on first divergence.

9. Canary Node [phase3_canary_node]  
   – Deployed to private test-net for 1-week continuous run.  
   – Exit: Zero mismatches and alert-free period.
