# Execution CSV Comparator

This directory contains tools for comparing execution consistency between embedded and remote execution modes.

## execution_csv_compare.py

A comprehensive tool for comparing CSV files generated by the execution consistency logging system.

### Features

- **Dual Join Strategy**: Primary join by `tx_id_hex`, fallback to `(block_num, tx_index_in_block)`
- **Field-by-Field Comparison**: Configurable comparison of execution result fields
- **Multiple Report Formats**: Summary text, detailed CSV, and JSON for programmatic analysis
- **Large File Support**: Streaming CSV processing for multi-GB files
- **Flexible Configuration**: Selectable comparison fields and options

### Usage

#### Basic Comparison
```bash
python3 scripts/execution_csv_compare.py \
  --left output-directory/execution-csv/20250828-143022-a1b2c3d4-EMBEDDED-EMBEDDED.csv \
  --right output-directory/execution-csv/20250828-143555-b2c3d4e5-REMOTE-REMOTE.csv \
  --output reports/
```

#### Skip Return Data (Faster)
```bash
python3 scripts/execution_csv_compare.py \
  --left embedded.csv \
  --right remote.csv \
  --output reports/ \
  --ignore-return-data
```

#### Compare Specific Fields Only
```bash
python3 scripts/execution_csv_compare.py \
  --left embedded.csv \
  --right remote.csv \
  --output reports/ \
  --fields is_success energy_used state_digest_sha256
```

### Input Files

The tool expects CSV files generated by the execution consistency logging system with the following schema:

- `tx_id_hex`: Primary join key (hexadecimal transaction ID)
- `block_num`, `tx_index_in_block`: Fallback join key
- `is_success`: Boolean execution success indicator
- `result_code`: Contract execution result code  
- `energy_used`: Energy consumed by transaction
- `return_data_hex`: Hexadecimal return data from execution
- `runtime_error`: Error message if execution failed
- `state_digest_sha256`: SHA-256 digest of state changes
- `state_change_count`: Number of state changes captured
- Plus additional context fields (block info, contract type, etc.)

#### 2. Detailed Mismatches (`mismatches.csv`)
CSV file with one row per mismatch, including:
- Join information (how the transaction was matched)
- Context (block number, transaction index, contract type)
- Side-by-side field values for all mismatched fields
- Execution mode information

Useful for detailed analysis of specific discrepancies.

#### 3. JSON Report (`comparison_results.json`)
Complete structured data including:
- All statistics and configuration
- Full mismatch details with nested structure
- Suitable for programmatic analysis or further processing

### Comparison Logic

#### Field Types

- **Exact Fields**: Must match exactly (is_success, result_code, return_data_hex, runtime_error, state_digest_sha256)
- **Numeric Fields**: Compared as integers (energy_used, state_change_count)
- **Hex Fields**: Normalized to lowercase before comparison

#### Join Strategy

1. **Primary Join**: Match records by `tx_id_hex`
2. **Fallback Join**: For unmatched records, attempt match by `(block_num, tx_index_in_block)`
3. **Missing Records**: Track transactions that exist in only one file

#### Normalization

- Boolean fields converted to standard format
- Hex fields normalized to lowercase
- Numeric fields parsed as integers
- String fields trimmed of whitespace

### Performance

The tool is designed to handle large CSV files efficiently:

- Streaming CSV processing (memory usage independent of file size)
- Indexed lookups for O(1) join performance  
- Progress reporting for long-running comparisons
- Configurable field selection to skip expensive comparisons

### Exit Codes

- `0`: Comparison completed successfully
- `1`: Error (file not found, invalid format, etc.)

### Examples

#### Complete Workflow
```bash
# Generate embedded execution CSV
java -jar FullNode.jar -c config.conf \
  -Dexec.csv.enabled=true \
  -Dstorage.mode=EMBEDDED \
  -Dexecution.mode=EMBEDDED

# Generate remote execution CSV  
java -jar FullNode.jar -c config.conf \
  -Dexec.csv.enabled=true \
  -Dstorage.mode=REMOTE \
  -Dexecution.mode=REMOTE

# Compare results
python3 scripts/execution_csv_compare.py \
  --left output-directory/execution-csv/*EMBEDDED*.csv \
  --right output-directory/execution-csv/*REMOTE*.csv \
  --output comparison-reports/

# Review results
cat comparison-reports/comparison_summary.txt
less comparison-reports/mismatches.csv
```

#### CI Integration
```bash
#!/bin/bash
# Compare execution modes and fail if too many mismatches

python3 scripts/execution_csv_compare.py \
  --left embedded.csv --right remote.csv --output reports/ \
  --ignore-return-data

# Parse JSON results to check mismatch rate
mismatch_rate=$(python3 -c "
import json
with open('reports/comparison_results.json') as f:
    data = json.load(f)
matched = data['statistics']['matched'] 
total = max(data['statistics']['left_total'], data['statistics']['right_total'])
if total > 0:
    print(100 * (total - matched) / total)
else:
    print(0)
")

if (( $(echo "$mismatch_rate > 5.0" | bc -l) )); then
    echo "ERROR: Mismatch rate ${mismatch_rate}% exceeds threshold"
    exit 1
fi

echo "SUCCESS: Mismatch rate ${mismatch_rate}% within acceptable range"
```

### Troubleshooting

#### Common Issues

1. **File Not Found**
   - Verify CSV file paths are correct
   - Check that execution logging generated the expected files

2. **No Matches Found**
   - Verify both CSV files contain data for the same block range
   - Check that tx_id_hex fields are properly formatted
   - Ensure block_num and tx_index_in_block are valid integers

3. **Memory Issues with Large Files**
   - The tool uses streaming processing, but very large output reports may consume memory
   - Consider using `--ignore-return-data` for files with large return payloads
   - Filter to specific fields with `--fields` to reduce memory usage

4. **Performance Issues**
   - Large files (>1GB) may take several minutes to process
   - Consider sampling the input data or processing smaller block ranges
   - Use `--ignore-return-data` to skip expensive hex comparisons

#### Debugging

Enable verbose output with Python logging:
```bash
PYTHONPATH=. python3 -u scripts/execution_csv_compare.py \
  --left embedded.csv --right remote.csv --output reports/ 2>&1 | tee comparison.log
```
